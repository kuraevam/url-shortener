## Результат вехи
Проект инициализирован, структура пакетов создана, `cmd/shorten` запускается и парсит аргументы: `shorten <longURL>` печатает короткий URL (заглушка), `shorten --resolve <code>` восстанавливает URL из временной `map`, `shorten --help`/`-h` печатает справку. Бизнес-логика `Shorten`/`Resolve` живёт в `internal/shortener` на заглушке-хранилище. Это базовый рабочий цикл, который вехи M2–M5 переведут на полноценное хранилище, генерацию кода, TTL и ошибки.

## Что собираем (шаг к результату)

### 1. Проверить окружение
- Убедиться, что установлен Go ≥ 1.24: `go version`.
- При необходимости установить/обновить Go и настроить IDE (GoLand или VS Code с расширением `golang.go`).
- Проверить переменные окружения: `go env GOROOT GOPATH GOBIN`.

### 2. Инициализировать модуль
- В корне репозитория выполнить:
  ```bash
  go mod init github.com/<user>/url-shortener
  ```
- В `go.mod` указать актуальную версию языка через директиву `go 1.24` (или новее).

### 3. Создать структуру пакетов (по [00-spec.md](00-spec.md), раздел 3.1)
Воспроизвести дерево:
```
cmd/shorten/main.go
internal/shortener/shortener.go
internal/shortener/codegen.go
internal/shortener/validate.go
internal/store/store.go
internal/store/memory.go
internal/store/link.go
internal/store/errors.go
internal/store/iter.go
internal/xslices/xslices.go
```
Пустые файлы создаются с заглушками `package <name>` и комментарием пакета. Файлы `codegen.go`/`validate.go`/`memory.go`/`link.go`/`errors.go`/`iter.go`/`xslices.go` наполняются в последующих вехах (M2–M6); на данном этапе достаточно объявить пакет и doc comment.

### 4. `cmd/shorten/main.go` (парсинг аргументов)
- `package main`.
- Использовать пакет `flag` для объявления флага `--resolve` (строка, по умолчанию пустая) и флага `--help`/`-h` (булев).
- Использовать `flag.Parse()`; остаток аргументов — `flag.Arg(0)` — длинный URL.
- Логика:
  - Если задан `--help` или `-h` → напечатать краткую справку (см. п. 4a) и выйти с кодом 0. При этом если `flag.Arg(0)` пуст и `--resolve` пуст (нет аргументов вообще) → напечатать справку и выйти с ненулевым кодом.
  - Если задан `--resolve <code>` → вызвать `Resolve(code)` и напечатать оригинальный URL.
  - Иначе если задан `flag.Arg(0)` → вызвать `Shorten(url)` и напечатать результат (полный короткий URL).
- Обрабатывать ошибки: выводить `error: <сообщение>` в `stderr`, `os.Exit(1)`.

### 4a. Справка `--help` / `-h`
Реализовать через стандартный механизм `flag` (завести булев флаг `-help`/`-h` или проверять `os.Args` вручную) и печатать:
```
Usage:
  shorten <longURL>            Создать короткий код для longURL, напечатать https://s.io/<code>
  shorten --resolve <code>    Напечатать оригинальный URL по короткому коду
  shorten --help | -h          Показать эту справку
```
Спека: раздел 4.1 [00-spec.md](00-spec.md) требует `shorten --help` / `shorten -h`.

### 5. Функция `Shorten` (в `internal/shortener/shortener.go`)
```go
// Shorten возвращает полный короткий URL (https://s.io/<code>) для longURL.
// В данной вехе код генерируется упрощённо (заглушка); полноценная
// генерация — в вехе M3.
func Shorten(longURL string) (string, error)
```
- Возвращает **полный короткий URL** `urlPrefix + code` (константа `urlPrefix = "https://s.io/"` из п. 7), а не голый код. Соответствует спеке 4.1.
- Валидация URL делегируется функции `Validate` (полная реализация — веха M3, здесь — минимальная проверка `strings.HasPrefix(url, "http://") || strings.HasPrefix(url, "https://")`). На данном этапе `Validate` объявляется в `shortener.go`; в вехе M3 переносится в `validate.go`.
- Временная генерация кода: например, `strconv.FormatInt(int64(len(url)), 36)` — заглушка, заменится в вехе M3.
- Возвращает ошибку при невалидном URL (в `main` выводится как `error: invalid URL` — см. веху M5).
- **Сохраняет маппинг** в пакетную переменную `store` (см. п. 6): `store[code] = longURL`. Без этого `Resolve` (и smoke-тест `--resolve`) не сможет восстановить URL.

### 6. Функция `Resolve` (в `internal/shortener/shortener.go`)
```go
// Resolve возвращает оригинальный URL по короткому коду.
// Пока использует глобальную map-переменную (заглушка); полноценное
// хранилище — веха M2.
func Resolve(code string) (string, error)
```
- Использует временную `var store = map[string]string{}` (объявление переменной-карты).
- Возвращает ошибку при отсутствии кода (здесь — `errors.New("not found")`; кастомные ошибки — веха M5).

### 7. Константы и `iota` (в `internal/shortener/shortener.go`)
```go
const (
    defaultCodeLen = 6
    urlPrefix      = "https://s.io/"
)

const (
    schemeHTTP schemeKind = iota
    schemeHTTPS
    schemeOther
)
```
Тип `schemeKind` — целочисленный; использовать `iota` для последовательных значений.

### 8. Документирование пакетов
- Каждый пакет снабдить doc comment над строкой `package`:
  ```go
  // Пакет shortener реализует логику сокращения и восстановления URL.
  package shortener
  ```
- Имена пакетов — короткие, в нижнем регистре, без `_` и `camelCase`.
- Doc comments на все экспортируемые идентификаторы.

## Отработка тем (практика и демо)

### Функции как значения и замыкания (в `internal/shortener/codegen.go`)
- Завести тип функции-генератора кода:
  ```go
  type CodeGenerator func(seed string) string
  ```
- Реализовать функцию `newCounterGenerator(start int) CodeGenerator`, возвращающую замыкание, инкрементирующее счётчик при каждом вызове (демонстрация closure).
- Использовать `CodeGenerator` в `Shorten` (пока — через дефолтный генератор).

### Вариативные функции (в `internal/shortener/shortener.go`)
- Реализовать `joinURLs(parts ...string) string` — объединяет части URL через `/`, демонстрируя variadic-параметр и `...` при передаче среза.

## Подсказки / подводные камни
- `internal/` недоступен для импорта извне модуля — это намеренно; не пытайтесь обойти.
- Имя модуля в `go mod init` лучше делать в формате `github.com/<user>/<repo>`, чтобы будущие главы могли добавлять зависимости без переименования.
- Не смешивайте `package main` и обычные пакеты в одном каталоге.
- `gofmt` проверяет только форматирование; `go vet` — подозрительные конструкции. Запускайте оба.
- `flag.Parse()` по умолчанию останавливается на первом не-флаговом аргументе; учитывайте порядок `--resolve <code>` vs позиционного URL.
- Замыкание захватывает переменную по ссылке, не по значению — счётчик корректно инкрементируется.
- `iota` сбрасывается в каждом `const`-блоке.
- Не объявляйте переменные «про запас» — Go требует использования всех объявленных переменных.
