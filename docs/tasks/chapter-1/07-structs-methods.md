## Цель
Оформить модель данных проекта через структуры `Link` и `Visit` с методами, продемонстрировать встраивание, выбор ресивера (value vs pointer), теги и правила экспортируемости.

## Что нужно сделать

### 1. Структура `Link` (`internal/store/link.go`)
```go
// Link описывает сокращённую ссылку и её метаданные.
type Link struct {
    Code      string
    LongURL   string
    CreatedAt time.Time
    TTL       time.Duration
    Hits      int64       // счётчик переходов (value-семантика для чтения)
}
```
- Инициализация в `Shorten` через литерал: `&Link{Code: code, LongURL: longURL, CreatedAt: time.Now(), TTL: DefaultTTL}`.
- Демонстрация инициализации через `new(Link)` и именованные поля — в smoke-функции.

### 2. Методы `Link` с разными ресиверами
- Pointer-receiver (мутирует состояние):
  ```go
  // IncHits увеличивает счётчик переходов. Pointer-receiver, т.к. мутирует.
  func (l *Link) IncHits() { l.Hits++ }
  // IsExpired — pointer-receiver согласован с другими методами Link.
  func (l *Link) IsExpired(at time.Time) bool
  ```
- Value-receiver (не мутирует):
  ```go
  // String возвращает человекочитаемое представление. Value-receiver.
  func (l Link) String() string
  ```
- Комментарий: объяснить правило — если хоть один метод требует pointer-receiver, все методы типа получают pointer-receiver для согласованности.

### 3. Встраивание (embedding)
- Завести тип `Audit`:
  ```go
  // Audit хранит служебные метаданные создания/изменения.
  type Audit struct {
      CreatedAt time.Time
      UpdatedAt time.Time
  }
  ```
- Встроить `Audit` в `Link`:
  ```go
  type Link struct {
      Code    string
      LongURL string
      TTL     time.Duration
      Audit   // встраивание — поля Audit промоутятся
  }
  ```
- Демонстрация: `link.CreatedAt` доступен напрямую (промоутинг полей); `link.Audit.UpdatedAt` — тоже.
- Метод `func (a *Audit) Touch()` — обновляет `UpdatedAt`; вызывается через `link.Touch()`.

### 4. Теги структур
- Добавить тег к полю `LongURL` (подготовка к JSON в Главе 3):
  ```go
  LongURL string `json:"long_url"`
  ```
- Демонстрация чтения тега через `reflect` — в задаче 08 не требуется; здесь только наличие тега и комментарий, что теги используются пакетами сериализации.

### 5. Указатели: семантика и подводные камни
- В `MemoryStorage` хранить `map[string]*Link` (указатели), чтобы изменения `Link` через методы были видны всем держателям ссылки.
- Демонстрационная функция `demoPointers()`:
  - Показать, что передача `Link` по значению копирует структуру (изменения не видны вызывающему).
  - Передача `*Link` — изменения видны.
  - Взятие адреса элемента map запрещено (`&m[k]`); для модификации — получить `link := m[k]`, мутировать, присвоить обратно, либо хранить `*Link`.
- Комментарий: nil-указатель на `Link` — `var l *Link; l.IncHits()` паникует; показывать безопасный вызов через `if l != nil`.

### 6. Экспортируемость
- Все публичные поля и методы — с заглавной буквы (`Code`, `LongURL`, `IncHits`).
- Приватные поля (если появятся) — со строчной; например, внутренний счётчик `hits int64` с публичным геттером `Hits() int64`.
- Демонстрация: пакет `internal/shortener` не может обратиться к приватному полю `internal/store`.

## Подсказки / подводные камни
- Нельзя взять адрес элемента map: `&m[k]` — ошибка компиляции; причина — map может переаллоцировать backing-таблицу, адрес стал бы невалидным.
- Смешивание value- и pointer-receiver для одного типа — плохая практика (мешает реализации интерфейсов, задача 08).
- Встраивание — это композиция, а не наследование; нет полиморфизма, только промоутинг полей и методов.
- `nil`-указатель можно вызвать для метода с pointer-receiver; метод обязан сам проверить `l == nil`, если это возможно.
- Копирование структуры с мьютексом через value-receiver — ошибка (`go vet` поймает `copylocks`); мьютексы появятся в Главе 2.
- Теги — строки, не проверяются компилятором; опечатки обнаруживаются только потребителем (например, `encoding/json`).
