## Цель
Реализовать каркас бизнес-логики `internal/shortener`: функции `Shorten(longURL string) (string, error)` и `Resolve(code string) (string, error)` с использованием базового синтаксиса Go. Парсинг аргументов в `cmd/shorten/main.go` через `os.Args` и пакет `flag`.

## Что нужно сделать

### 1. Парсинг аргументов CLI (`cmd/shorten/main.go`)
- Использовать пакет `flag` для объявления флага `--resolve` (строка, по умолчанию пустая).
- Использовать `flag.Parse()`; остаток аргументов — `flag.Arg(0)` — длинный URL.
- Логика:
  - Если задан `--resolve <code>` → вызвать `Resolve(code)` и напечатать URL.
  - Иначе если задан `flag.Arg(0)` → вызвать `Shorten(url)` и напечатать `https://s.io/<code>`.
  - Иначе напечатать справку и вернуть ненулевой код.
- Обрабатывать ошибки: выводить `error: <сообщение>` в `stderr`, `os.Exit(1)`.

### 2. Функция `Shorten`
```go
// Shorten возвращает короткий код для longURL. В данной задаче код
// генерируется упрощённо (заглушка); полноценная генерация — в задаче 04.
func Shorten(longURL string) (string, error)
```
- Валидация URL делегируется функции `Validate` (полная реализация — задача 04, здесь — минимальная проверка `strings.HasPrefix(url, "http://") || strings.HasPrefix(url, "https://")`).
- Временная генерация кода: например, `strconv.FormatInt(int64(len(url)), 36)` — заглушка, заменится в задаче 04.
- Возвращает ошибку при невалидном URL.

### 3. Функция `Resolve`
```go
// Resolve возвращает оригинальный URL по короткому коду.
// Пока использует глобальную map-переменную (заглушка); полноценное
// хранилище — задача 05.
func Resolve(code string) (string, error)
```
- Использует временную `var store = map[string]string{}` (тема 1.2.2 — объявление переменных).
- Возвращает ошибку при отсутствии кода (здесь — `errors.New("not found")`; кастомные ошибки — задача 09).

### 4. Демонстрация управляющих конструкций
- В `Shorten`/`Resolve` использовать:
  - `if` с инициализатором: `if err := Validate(longURL); err != nil { ... }`.
  - `switch` без выражения (как `if-else` цепочка) — например, выбор префикса по схеме URL.
  - `for` с `range-over-int`: `for i := range count { ... }` в демонстрационной функции.
- Добавить в `internal/shortener` пакетную функцию `demoControlFlow()` (неэкспортируемая), вызываемую из теста или `main` для самопроверки — показывает `if`, `switch`, `for` (включая `range-over-int`).

### 5. Константы и `iota`
- Завести блок констант в `internal/shortener/shortener.go`:
  ```go
  const (
      defaultCodeLen = 6
      urlPrefix      = "https://s.io/"
  )

  const (
      schemeHTTP schemeKind = iota
      schemeHTTPS
      schemeOther
  )
  ```
- Тип `schemeKind` — целочисленный; использовать `iota` для последовательных значений.

### 6. Функции как значения и замыкания
- Завести тип функции-генератора кода:
  ```go
  type CodeGenerator func(seed string) string
  ```
- Реализовать функцию `newCounterGenerator(start int) CodeGenerator`, возвращающую замыкание, инкрементирующее счётчик при каждом вызове (демонстрация closure).
- Использовать `CodeGenerator` в `Shorten` (пока — через дефолтный генератор).

### 7. Вариативные функции
- Реализовать `joinURLs(parts ...string) string` — объединяет части URL через `/`, демонстрируя variadic-параметр и `...` при передаче среза.

## Подсказки / подводные камни
- `flag.Parse()` по умолчанию останавливается на первом не-флаговом аргументе; учитывайте порядок `--resolve <code>` vs позиционного URL.
- Замыкание захватывает переменную по ссылке, не по значению — счётчик корректно инкрементируется.
- `iota` сбрасывается в каждом `const`-блоке.
- Не объявляйте переменные «про запас» — Go требует использования всех объявленных переменных.
- `range-over-int` (`for i := range n`) доступен с Go 1.22+.
