# Задача 01. Инициализация проекта и окружение

## Связанные темы
- 1.1.1 Введение в Go (история, ключевые особенности, релизный цикл)
- 1.1.2 Установка Go, настройка окружения и выбор IDE
- 1.1.3 Структура первой программы на Go
- 1.1.4 Инструменты разработки: go CLI, gofmt, go vet
- 1.3.1 Effective Go и правила именования
- 1.3.2 Документирование кода: doc comments, godoc
- 1.11.1 Пакеты: организация кода и правила именования
- 1.11.2 Go-модули: go.mod, go.sum, версионирование

## Цель
Подготовить рабочее окружение, инициализировать Go-модуль и создать скелет проекта согласно структуре из [00-spec.md](00-spec.md) (раздел 3.1). Получить собираемую «Hello, World»-программу `cmd/shorten`, проходящую `gofmt` и `go vet`.

## Что нужно сделать

### 1. Проверить окружение
- Убедиться, что установлен Go ≥ 1.24: `go version`.
- При необходимости установить/обновить Go и настроить IDE (GoLand или VS Code с расширением `golang.go`).
- Проверить переменные окружения: `go env GOROOT GOPATH GOBIN`.

### 2. Инициализировать модуль
- В корне репозитория выполнить:
  ```bash
  go mod init github.com/<user>/url-shortener
  ```
- В `go.mod` указать актуальную версию языка через директиву `go 1.24` (или новее).

### 3. Создать структуру пакетов
Воспроизвести дерево из [00-spec.md](00-spec.md), раздел 3.1:
```
cmd/shorten/main.go
internal/shortener/shortener.go
internal/store/store.go
internal/store/memory.go
internal/store/link.go
internal/store/iter.go
```
Пустые файлы можно создать с заглушками `package <name>` и комментарием пакета.

### 4. Написать `cmd/shorten/main.go`
- `package main`.
- Функция `main()` печатает `shorten: ready` в `stdout`.
- Doc comment на `main` не требуется (неэкспортируемая), но на `package main` — опционально.

### 5. Документирование пакетов
- Каждый пакет снабдить doc comment над строкой `package`:
  ```go
  // Пакет shortener реализует логику сокращения и восстановления URL.
  package shortener
  ```
- Имена пакетов — короткие, в нижнем регистре, без `_` и `camelCase`.

### 6. Проверить инструменты
- `go build ./...` — без ошибок.
- `go vet ./...` — без замечаний.
- `gofmt -l .` — пустой список (все файлы отформатированы).
- `go run ./cmd/shorten` — печатает `shorten: ready`.

## Требования к коду
- Именование по Effective Go: `MixedCaps`, пакеты — короткие строчные.
- Комментарии — на русском, имена идентификаторов — на английском.
- Все экспортируемые идентификаторы (на данном этапе их мало) с doc comments.
- Внешних зависимостей не добавлять.

## Критерии готовности
- [ ] `go.mod` инициализирован, `go 1.24+` указан.
- [ ] Структура каталогов из раздела 3.1 [00-spec.md](00-spec.md) создана.
- [ ] `cmd/shorten/main.go` собирается и запускается.
- [ ] `go build ./...`, `go vet ./...`, `gofmt -l .` проходят чисто.
- [ ] Каждый пакет имеет doc comment.

## Оценка времени
~6 часов (включая установку/настройку окружения при необходимости).

## Подсказки / подводные камни
- `internal/` недоступен для импорта извне модуля — это намеренно; не пытайтесь обойти.
- Имя модуля в `go mod init` лучше делать в формате `github.com/<user>/<repo>`, чтобы будущие главы могли добавлять зависимости без переименования.
- Не смешивайте `package main` и обычные пакеты в одном каталоге.
- `gofmt` проверяет только форматирование; `go vet` — подозрительные конструкции. Запускайте оба.
